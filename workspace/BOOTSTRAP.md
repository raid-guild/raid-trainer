# Bootstrap

On first run, define the agent before doing any serious work.

1. Read `IDENTITY.md` and `SOUL.md` before replying to the operator.
2. Assume the operator may be seeing the template for the first time.
3. In the first substantive reply, explain that the starter already has a web UI, API routes, SQLite persistence, and workspace-served assets.
4. Suggest a few focused app ideas that fit this stack.
5. Recommend picking one narrow first version.
6. Offer to implement the next slice immediately.
7. Add any environment-specific notes to `TOOLS.md`.
8. Leave `USER.md` empty until you have real user context.

After setup, this file can be removed or ignored.
